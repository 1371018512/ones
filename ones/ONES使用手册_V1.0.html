序言

概述

关于

ONES是由诸城市特思维软件集成开发有限公司开发的企业管理软件，致力于为中小企业提供客户管理、仓储管理、财务管理等信息化服务。

ONES是基于模块化的核心+扩展的架构，企业可以根据自身需要来安装不同的模块组合。目前最新版本为1.0版本，囊括了客户关系管理，进销存管理，供应商管理，统计分析等模块。

本篇文档针对1.x之后版本所写。

使用方式

您可以通过使用ONES提供的公有云服务或者自己搭建私有云环境来部署使用，这取决于您的应用场景及网络环境。

开始使用

安装

使用ONES提供的公有云服务

使用ONES提供的公有云服务不需要用户自己安装，仅需要通过注册向导注册即可，这也是我们推荐使用的方式；而且很多模块需要外网使用环境，比如移动端模块、短信/邮件等提醒模块。

使用公有云服务也无需担心数据安全和访问速度，我们每天都有数据备份，防止数据丢失；国内访问速度和本地几乎无差别。

访问地址：https://ones-cloud.com

自行部署使用

如果您公司有技术人员，或者有一定的动手能力，或者因为其他原因需要部署到私有云使用，请参见开发文档中 安装与升级 一节；也可以联系我们付费进行安装。

初次使用配置

设置用户角色

安装或注册后，系统会自动生成一个名为「Super User」的超级管理角色，初次使用时请先为其授权。依次点击「系统首选项-》用户角色管理-》右键点击角色名称-》点击授权」，根据需要选定其可用权限，并保存(初次使用建议全部选择)。

启用应用

授权完成后，刷新页面，依次点击「系统首选项-》应用管理」，根据需要启用相应的应用。

工作流设置

如果使用到工作流（通常所有单据类应用都会使用到，比如采购、出入库、销售等），需要在「系统首选项-》工作流」中创建工作流程，具体可参见「工作流」一章。

系统首选项设置

依次点击「系统首选项-》首选项」可对系统进行设置。

界面介绍

页面元素

  • 上方为一级导航菜单
  • 右上角为个人菜单
  • 左侧为二级导航菜单，点击一级导航进行切换
  • 中间部分为页面主体
  • 一级导航和主体页面之间为标签页切换部分

数据展现与录入形式

ONES的数据展现默认可通过「数据表格」和「数据详情」两种方式展示。
数据录入包括「表单」和「单据」两种方式。单据即为入库单、采购单、销售单等。

数据表格

[图2]
表格最左侧复选框为多选操作，一些操作(如删除项目)可支持选中多个项目进行操作，点击复选框选中后，点击主体部分左上方的选中项目可看到可用操作；如果对单个项目进行操作还可在项目「行」上面点击右键。
表格右侧为过滤器，可通过点击的方式快速对数据进行过滤；如果模块支持，「列」最下方会有一个输入框，用于对当前列数据进行搜索；表格右下方为数据分页。
在表格「行」上面右键会显示当前行项目可用操作，双击左键会进入项目的详情界面或者编辑界面（取决于模块是否支持）。

数据详情

[图3]
详情页面通常用于展示复杂数据，如客户信息；客户信息可能包含很多项目，比如客户基本信息，客户联系人，客户跟踪情况等等等等，这种情况下就需要有一个页面对其所有信息进行总览；如果模块支持数据详情功能，在数据表格行上面双击可进入详情界面。
详情界面左侧为详细数据展示区域，右侧为详情项目导航区域，点击项目导航可切换展示内容。

表单

[图4]
表单是录入数据的方式之一，用于单条数据录入，如用户信息、客户基本信息等等。录入数据会进行三次验证，一是输入时即时验证，二是点击提交时前端验证，三是提交至后台处理时验证。
表单中支持多种「输入控件」，如文本输入框、数字输入框、邮件输入框、单选/复选输入、选择输入框、文件上传等等，这些控件有很多在单据中也支持。

单据

[图5]
单据包括三个部分，上面为基本信息区，从左至右为单据名称（可编辑），公司信息（系统首选项中设置），单据编号/二维码；中间为单据明细录入区；最下方为单据其他信息和统计区域，如录入时间、选择客户、合计等。
明细区域左侧为单据行号和增删行按钮，点击加号可增加一行，垃圾桶可删除当前行。点击明细行中的单元格可录入数据。

控制面板

控制面板是用户登入系统后第一个界面，也是默认不可被关闭的标签页。

[图1]

控制面板中可以添加各个应用提供的「控制面板控件」，它们可被拖动改变位置及缩放；如果没有任何控件被启用，则在页面中间会显示比较醒目的新增控件按钮；如果有任何已启用的控件，则新增控件按钮被移至主体部分右上角。

[图二]

点击新增控件按钮，在弹出的modal中选择需要添加的控件，并点击确定；然后拖动控件的边缘可调整大小，左键点击并按住控件任意部分可拖动位置。

权限控制

ONES中包含多种权限控制手段，通过各种组合可实现基本访问、工作流等灵活的权限控制。

基本操作认证 扩展的RBAC

RBAC(Role-Based Access Control)即「基于角色的访问控制」，在RBAC中，权限与角色相关联，用户通过成为适当角色的成员而得到这些角色的权限。这就极大地简化了权限的管理。在一个组织中，角色是为了完成各种工作而创造，用户则依据它的责任和资格来被指派相应的角色，用户可以很容易地从一个角色被指派到另一个角色。角色可依新的需求和系统的合并而赋予新的权限，而权限也可根据需要而从某角色中回收。角色与角色的关系可以建立起来以囊括更广泛的客观情况。

ONES中对角色的概念进行了扩展，配合「部门」对于可授权的节点加入了「标记(flag)」的概念；模块中的授权节点被添加标记后，可为其增加更细化的「行级权限控制」，标记可选择仅自己部门、仅自己和下属等人性化的授权方式。

[图1 授权界面截图]

工作流程中可执行的人、角色或部门

在ONES的工作流设计中，用户可设定某个操作为「角色」、「部门」、「用户」等任意颗粒度的可执行权限；具体可参照「工作流」一节。

个人设置

个人资料

用户可自行修改自己的部分资料，如姓名，电话等等，点击页面右上角的用户头像-》个人资料进行设置。

「个人资料支持自定义字段」

个人首选项

每个系统用户可对自己进行个性化配置，点击页面右上角的用户头像-》个人首选项进行设置。

数据模型 (自定义字段)

数据模型是ONES的特色功能之一，可为任意模块添加自定义的字段，并可为其设置是否可搜索，是否在列表中显示，输入控件等配置项。

基本使用

模块中支持数据模型的已设定，用户只需通过「基础数据-》数据模型字段」中添加即可。
[图1 数据模型字段列表]

点击「新增数据模型字段」按钮可添加新的字段，需要填写以下表单：
[图2 新增模型字段]

  • 从属于模块 选择需要添加字段至某模块
  • 别名 必须为英文，通常为程序内部使用，不会被显示
  • 显示名称 为字段在页面中显示的名称
  • 输入控件 选择字段的输入方式，注意单选、复选、下拉框、项目选择、select3等字段必须配置「data_source」项目
  • 数据类型 通常输入控件为数字、项目选择、select3的需要选择数字，单选、复选、下拉框等根据具体的数据源选择(value是否为数字)。
  • 是否可搜索 选择字段是否可在列表中进行搜索
  • 扩展配置见下表

扩展配置

模型字段扩展配置会默认覆盖可选择的配置(search_able, widget, data_type等)
扩展配置语法为：key:value 每行一个。支持列表如下 前端数据API中的config.fields中支持的配置均可使用：
~
list_display:true # 是否在列表中显示 默认 false
sortable:true # 是否可排序 默认false
required:false # 是否必填，默认 true
widget:select # 输入控件 默认 text
data_source:[1,2,3] # 数据源。 可指定为数组或前端数据API。 数组最好不要使用关联数组（JS中的对象）
filters:link # 加入过滤器
blank:true # 对应required，是否可为空。默认为false
~

应用模块

工作流

在企业的生产经营过程中，任何一项事务或业务都是有着流程存在的；传统的企业管理软件存在着流程固化或者流程设置麻烦的问题；ONES 1.0版本开始提供了一套可视化的流程设置系统，业务流程设置方便而又快速。并且流程可设置多个供业务过程中选择。

哪些模块支持工作流

目前所有单据类的模块支持工作流，如订单，采购，出入库等。

创建工作流

工作流设置位于「系统首选项-》工作流」中，点击新增。
[图1 创建工作流]

工作流模块字段是需要手动填写的，格式为 应用alias.应用模块alias， 比如：storage.stockIn； 可以在具体模块的文档中查询到。

工作流设计

[图2 工作流设计]
在工作流列表中，点击右键可显示工作流设计按钮，点击工作流设计进入可视化工作流设计界面。

设计界面中上方为工具栏，从左至右为「添加新的节点」，「新节点添加至哪个执行者」，「新增执行者」

节点： 工作流中的一个操作节点，用于执行设置好的业务逻辑
执行者： 节点容器，可为「用户」、「角色」、「部门」
开始节点：每个工作流有且必有一个开始节点

节点双击可设定其属性（或者右键单击节点，点击节点设置）
[图3 工作流节点设置]

节点状态标签：显示标签，标识当前流程数据进行至的状态
节点动作类型：可选「执行一个服务接口」或者「修改当前流程数据」

右键点击节点，可显示节点操作菜单：
下一节点是： 执行完当前节点后下一个可执行节点
条件为真/假： 条件判断节点专用

点击节点或者连线可将其设为活动状态，按「delete」键可删除。
